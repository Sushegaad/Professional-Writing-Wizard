① User Input
  • User pastes or types text into the left textarea
  • oninput fires live word count updates
  • Mode toggle updates currentMode variable
  • Clicking "Enhance" triggers the API call
② Fetch / API Call
  • POST to api.anthropic.com/v1/messages
  • System prompt encodes mode-specific instructions
  • Model: claude-sonnet-4-20250514
  • max_tokens: 2000 · Returns JSON array
③ Claude Inference
  • Analyses text against writing guidelines
  • Identifies 5–10 highest-impact changes
  • Returns structured JSON with original, improved, reason, category
  • Each original must match exactly for diff to work
④ Response Parsing
  • Strip markdown code fences if present
  • JSON.parse() with try/catch fallback to [ ]
  • Positioned by indexOf() in original text
  • Sorted by position for correct diff rendering
⑤ State Management
  • Pure in-memory JS — no localStorage
  • acceptedChanges / declinedChanges are ES6 Sets
  • Re-render triggered on every accept/decline action
  • All state resets on "Clear" or new enhancement
⑥ Diff Rendering
  • Segments split by change position, text, and state
  • Pending changes rendered as clickable green spans
  • Accepted = locked green; Declined = reverts to original
  • Copy builds final text by applying accepted changes
